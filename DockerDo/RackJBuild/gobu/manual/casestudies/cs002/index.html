Case Studies - Inparanoid

You may omit Examples for skipping operational content.
  • Inparanoid is a program that produces orthologous groups of two given species. In this case study, we use NCBI refseq human (build 33) and mouse (build 30) proteins as input. The output of the Inparanoid program is in a hierarchical structure; that is, input proteins are partitioned into orthologous groups, and there are human sub-group and mouse sub-group in each orthologous group.
  • Such a hierarchical output data can be easily organized as GOBU input. For each orthologous group, we use a user node "tax9606" to represent the human sub-group, and "tax10090" for mouse sub-group. To present the special relationship of each orthologous group, we assign each orthologous group node as a unit node. Every unit node will be shown with a "U" icon.

    • Example: Select File -> Load, open "inparanoidTax9606Tax10090.tree" file, the human-mouse orthologous data will be loaded.
  • We also download GO annotations and location information from NCBI database and assign these annotations to refseq proteins. Here we use the LOC property for representing location information.
  • Under GOBU's extendable architecture, we built an extension software (plugin) called ConservedSegmentView. ConservedSegmentView enables biologists to brwose orthologous relationships (in terms of proteins) in conserved segment pairs.

    As shown in the figure, the two black bars are a human chromosome segment and a mouse chromosome segment, which are curated as a syntenic block by the UM-mathod. Red boxes are protein locations (according to LocusLink table loc2ref and NCBI MapViewer data). A gray line linking two red boxes means an orthologous relationship curated by Inparanoid. We expect to see colinear gray lines in all conserved segment pairs.
